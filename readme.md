файл room.go

Функция newRoom():
  Возвращает структуру для комнаты чата.

Функция Run():
  Содержит цикл, который должен быть запущен в отдельной горутине.
  Отслеживает три канала:
	join - присоединение клиентов
	leave - отсоединение клиентов
	forward - поступление сообщений
  Каналы join и leave нужны чтобы избежать проблемы одновременного доступа к Карте clients.
Карта clients - содержит подключенных клиентов. Если клиент подключается - он добавляется в
карту, если ему не удалось доставить сообщение - он удаляется из карты.
  Если приходит сообщение в канал forward оно перенаправляется всем клиентам.

Функция ServeHTTP():
  Создает структуру клиента и передает его в канал join для присоединения к чату в данной
комнате.
  Передает клиента в канал leave при отключении клиента.
  Запускает метод write в отдельной горутине
  Запускает метод read в главной нити, который не дает ей завершиться т.е. держит соединение.
  См. описание методов write и read для файла сlient.go

файл сlient.go

метод read():
  Позволяет клиенту читать из сокета и пересылать сообщения в канал forward в комнату.
метод write():
  Получает сообщения из канала send и пересылает в сокет.

файл main.go

Функция main():
  Выводит в консоль временный каталог куда будут загружаться файлы.
  Создает комнату чата.
  Назначает обработчики для корневого каталога("\") и комнаты("\room")
  Запускает комнату в отдельной горутине
  Прослушивает порт 8080
  
Функция ServeHTTP():
  Загружает html из файла шаблона, компилирует его и полученный html записывает в
ResponseWriter для отправки в браузер.
  Если Post запрос загружаем файл на серевер.
  Здесь для функции CopyBuffer указан размер буфера 250Mb как указано в
задании.
